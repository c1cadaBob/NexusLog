# 逻辑架构

## 概述

本文档描述 NexusLog 系统的逻辑架构，包括各层职责和组件划分。

## 分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         表现层 (Presentation)                    │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  Frontend Console (React + Ant Design + ECharts)        │    │
│  └─────────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────────┤
│                         网关层 (Gateway)                         │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  OpenResty (Nginx + Lua) - 路由、认证、限流              │    │
│  └─────────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────────┤
│                         服务层 (Services)                        │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐            │
│  │ Control Plane│ │ API Service  │ │ Data Services│            │
│  └──────────────┘ └──────────────┘ └──────────────┘            │
├─────────────────────────────────────────────────────────────────┤
│                         数据层 (Data)                            │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐            │
│  │Elasticsearch │ │ PostgreSQL   │ │ Redis Cluster│            │
│  └──────────────┘ └──────────────┘ └──────────────┘            │
└─────────────────────────────────────────────────────────────────┘
```

## 业务域划分

| 域 | 职责 | 核心组件 |
|-----|------|----------|
| 采集域 | 日志采集和预处理 | Collector Agent, Kafka |
| 处理域 | 实时计算和分析 | Flink, CEP |
| 存储域 | 数据持久化 | Elasticsearch, PostgreSQL, MinIO |
| 查询域 | 日志检索和分析 | Query API, Aggregation |
| 告警域 | 告警规则和通知 | Alert Engine, Notification |
| 安全域 | 认证授权审计 | Keycloak, OPA, Vault |
