# 数据流架构

## 概述

本文档描述 NexusLog 系统的数据流向，包括日志采集、处理、存储和查询的完整链路。

## 日志采集流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 应用服务    │───▶│ Log Agent   │───▶│   Kafka     │───▶│   Flink     │
│ (日志源)    │    │ (采集代理)  │    │ (消息队列) │    │ (流处理)   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                │
                                                                ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 用户查询    │◀───│ Query API   │◀───│Elasticsearch│◀───│ 索引写入    │
│ (前端)      │    │ (查询服务)  │    │ (搜索存储) │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 数据流详情

### 1. 采集阶段

| 步骤 | 组件 | 处理内容 |
|------|------|----------|
| 1.1 | Log Agent | 文件采集、格式解析、字段提取 |
| 1.2 | Log Agent | 数据压缩、批量发送 |
| 1.3 | Kafka | 消息持久化、分区路由 |

### 2. 处理阶段

| 步骤 | 组件 | 处理内容 |
|------|------|----------|
| 2.1 | Flink | 数据清洗、字段标准化 |
| 2.2 | Flink | 实时聚合、异常检测 |
| 2.3 | Flink | 告警规则匹配 |

### 3. 存储阶段

| 步骤 | 组件 | 处理内容 |
|------|------|----------|
| 3.1 | Elasticsearch | 全文索引、时序存储 |
| 3.2 | PostgreSQL | 元数据、配置存储 |
| 3.3 | MinIO | 冷数据归档 |

### 4. 查询阶段

| 步骤 | 组件 | 处理内容 |
|------|------|----------|
| 4.1 | Query API | 查询解析、权限校验 |
| 4.2 | Elasticsearch | 全文检索、聚合计算 |
| 4.3 | Redis | 结果缓存 |

## 数据保留策略

| 数据类型 | 热存储 | 温存储 | 冷存储 |
|----------|--------|--------|--------|
| 原始日志 | 7 天 | 30 天 | 1 年 |
| 聚合数据 | 30 天 | 90 天 | 永久 |
| 审计日志 | 90 天 | 1 年 | 7 年 |
