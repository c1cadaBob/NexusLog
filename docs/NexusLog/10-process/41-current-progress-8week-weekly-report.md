# NexusLog 基于当前完成内容的 8 周周报拆分（截至 2026-04-21）

## 1. 文档说明

本文档基于当前仓库中已经完成或已固化的功能、任务台账、实现代码与相关证据材料，对项目阶段性成果进行 8 周拆分整理。

编写原则如下：

1. 周期划分参考既定排期节奏。
2. 内容优先以当前仓库真实完成情况为准。
3. 每周统一包含以下两部分：
   - 本周所完成的主要内容及进度情况
   - 本周小结
4. 对于截至 `2026-04-21` 仍处于进行中的工作，采用“推进中”口径描述，不夸大为“全部完成”。

---

## 2. 8 周内容拆分

### 2026年3月02日-2026年3月08日

#### 本周所完成的主要内容及进度情况

1. 完成项目需求冻结，明确系统核心范围，聚焦认证鉴权、日志采集接入、检索分析、告警治理、审计追踪与可视化展示等主线模块。
2. 完成总体架构方案与接口主框架梳理，明确前端控制台、网关、认证服务、控制面、查询服务、审计服务、导出服务等模块边界。
3. 完成开发环境与联调基线搭建，建立容器化开发路径、热更新开发方式以及统一本地启动入口。
4. 完成数据库迁移真相源统一，明确迁移唯一入口，完成 `up/down` 演练与回滚验证，为后续功能落库提供稳定基础。
5. 初步实现认证与网关主干最小可用版本，为后续认证接口与前端登录主路径联调打通基础路径。
6. 本周整体项目进度建议记为约 `15%`。

#### 本周小结

本周完成了从方案规划到可执行开发基线的切换，项目已经具备持续开发与联调条件，M1 阶段启动前置条件基本完成。

### 2026年3月09日-2026年3月15日

#### 本周所完成的主要内容及进度情况

1. 完成认证闭环核心能力建设，覆盖注册、登录、刷新、退出、重置密码申请、重置密码确认等认证接口。
2. 完成认证相关数据落库与校验，包含会话、密码重置令牌、登录尝试记录等关键表写入与状态校验。
3. 统一认证接口响应结构、错误码与异常处理逻辑，提升前后端联调一致性。
4. 完成前端登录、注册、忘记密码、受保护路由等主路径改造，前端从本地模拟切换为真实 API 调用。
5. 同步修正网关白名单、认证转发规则与错误响应体，保证认证主路径在真实环境下连通。
6. 本周整体项目进度建议记为约 `28%`。

#### 本周小结

本周重点完成认证主路径闭环，前后端、网关与数据库之间已经形成稳定认证链路，M1 核心目标基本收口。

---

### 2026年3月16日-2026年3月22日

#### 本周所完成的主要内容及进度情况

1. 完成日志接入控制面主接口建设，覆盖拉取源管理、任务触发、任务列表查询、任务详情查询、包查询、回执写入与死信重放等关键接口。
2. 打通拉取任务状态流转，支持任务从 `pending -> running -> success | failed | canceled` 的状态变化与追踪。
3. 完成 Agent 侧系统指标上报基础能力，并在控制面提供指标接收与查询能力。
4. 建立资源阈值基础治理能力，为后续告警触发、通知派发与事故升级提供输入条件。
5. 完成控制面基础监控和延迟观测能力，具备对拉取链路关键指标进行最小化跟踪的能力。
6. 本周整体项目进度建议记为约 `42%`。

#### 本周小结

本周完成了控制面从接口框架到可执行能力的推进，接入控制与任务追踪链路已具备闭环基础，M2 正式进入可联调阶段。

---

### 2026年3月23日-2026年3月29日

#### 本周所完成的主要内容及进度情况

1. 完成 Agent 文件增量采集能力，支持基于 checkpoint 的断点续传与增量读取。
2. 实现 Agent 主动拉取相关接口，包括元信息获取、日志拉取与回执确认接口。
3. 完成 Agent API Key 鉴权能力，支持 `X-Agent-Key` 与 `X-Key-Id` 的校验机制，提升链路安全性。
4. 交付 Agent 双安装模式，覆盖预编译二进制 + systemd 与 Docker 运行模式，满足不同部署场景。
5. 打通 ACK/NACK、回执、失败补偿、死信重放等关键机制，确保接入链路具备最小可追溯与可恢复能力。
6. 将 Kafka 主推链路作为兼容项纳入，保留后续扩展能力，但不作为当前阶段 P0 验收前提。
7. 本周整体项目进度建议记为约 `58%`。

#### 本周小结

本周完成了 Agent 到控制面的关键接入闭环，采集、拉取、确认、补偿与追踪机制基本成立，M2 主链路已具备可验证的工程形态。

### 2026年3月30日-2026年4月05日

#### 本周所完成的主要内容及进度情况

2. 完成前端实时检索页、查询历史页、收藏查询页接入真实查询接口，逐步摆脱主路径 Mock 数据依赖。
3. 完成检索元数据读写能力建设，支持查询历史与收藏查询的持久化。
4. 建立基础统计与聚合展示能力，为首页概览、趋势分析与读侧扩展提供数据支撑。
5. 补充导出任务接口与聚合入口能力，使读侧从纯查询扩展到导出与概览聚合。
6. 本周整体项目进度建议记为约 `73%`。

#### 本周小结

本周检索主链路取得明显进展，查询能力已从页面形态进入真实数据驱动阶段，系统读侧主路径基本成型。
