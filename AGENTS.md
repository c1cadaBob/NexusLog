# AGENTS.md

## 前端调试强制规则

- 所有涉及前端页面内容的任务，必须使用 `chrome-devtools` MCP 工具进行调试与验证。
- 适用范围包括页面渲染、交互行为、路由跳转、页面接口联调、浏览器控制台报错。
- 每次结论必须包含四类证据：目标 URL、Console 信息、Network 请求、可复现步骤。
- 缺少上述任一证据的结论视为无效结论。
- 若当前会话无法使用 `chrome-devtools` MCP，任务应判定为阻塞并明确报告阻塞原因。

## Git 提交规则

- 每一轮对话只要产生了仓库文件变更，结束前必须执行一次 `git add -A` 与 `git commit`，确保改动进入版本控制。
- 纯讨论或未产生文件变更的回合，不要求创建空提交；如需严格保留每轮痕迹，可再改为允许空提交。
- 提交说明应简洁明确，优先概括本轮完成的核心改动。
